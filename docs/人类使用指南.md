## 这是什么？

ClawQuant（小龙虾量化）是一个命令行工具，帮你做三件事：

1. **从交易所拉数据** — 自动从 Binance 等交易所拉取历史K线数据
2. **回测交易策略** — 用历史数据模拟"如果当时按这个策略买卖，结果会怎样"
3. **生成分析报告** — 输出收益率、最大回撤、稳定性评分等专业指标

简单来说：**不花一分钱，先用历史数据验证你的交易想法是否靠谱。**

## 安装

### 方法 1：直接从 PyPI 安装（推荐）

打开终端（Mac 叫"终端"，Windows 叫"命令提示符"或"PowerShell"），输入：

```bash
pip install clawquant
```

装好后验证：

```bash
clawquant --version
# 应该输出: ClawQuant Trader v0.1.0
```

### 方法 2：从源码安装

```bash
# 1. 下载代码
git clone https://github.com/duolaAmengweb3/clawquant-trader.git
cd clawquant-trader

# 2. 创建虚拟环境（推荐，避免污染系统 Python）
python -m venv .venv
source .venv/bin/activate    # Mac/Linux
# .venv\Scripts\activate     # Windows

# 3. 安装
pip install -e .
```

### 需要代理？

如果你在国内，访问 Binance API 可能需要代理。创建一个 `.env` 文件：

```bash
cp .env.example .env
```

编辑 `.env`，填入你的代理地址：

```
HTTPS_PROXY=http://127.0.0.1:7897
```

## 第一步：拉取数据

回测之前，需要先把历史K线数据拉到本地。

### 最简单的用法

```bash
clawquant data pull BTC/USDT
```

这会拉取 BTC/USDT 最近 **10 天** 的 **1小时** K线数据。

### 拉更长时间

```bash
# 拉 30 天
clawquant data pull BTC/USDT --days 30

# 拉 180 天（半年）
clawquant data pull BTC/USDT --days 180

# 拉 365 天（一年）
clawquant data pull BTC/USDT --days 365
```

### 指定日期范围

```bash
# 拉 2025 年全年的数据
clawquant data pull BTC/USDT --start 2025-01-01 --end 2025-12-31

# 拉某个特定时间段
clawquant data pull BTC/USDT --start 2026-01-15 --end 2026-02-28
```

### 换不同的K线周期

```bash
# 日线（每根K线代表1天）— 适合看长期趋势
clawquant data pull BTC/USDT --interval 1d --days 365

# 4小时线 — 中期分析
clawquant data pull BTC/USDT --interval 4h --days 90

# 1小时线（默认）— 最常用
clawquant data pull BTC/USDT --interval 1h --days 30

# 15分钟线 — 短线分析
clawquant data pull BTC/USDT --interval 15m --days 7

# 5分钟线 — 超短线
clawquant data pull BTC/USDT --interval 5m --days 3
```

**可用周期**: `1m`（1分钟）、`5m`、`15m`、`30m`、`1h`（1小时）、`4h`、`1d`（1天）

### 同时拉多个币种

```bash
clawquant data pull BTC/USDT,ETH/USDT,SOL/USDT --interval 1h --days 30
```

### 查看已缓存的数据

```bash
clawquant data cache-status
```

### 检查数据质量

```bash
clawquant data inspect BTC/USDT --interval 1h
```

> **提示**: 数据会自动缓存到 `data_cache/` 目录，下次拉取相同数据会自动跳过已有的部分，速度很快。

## 第二步：跑一次回测

数据拉好了，现在用策略跑回测。

### 最简单的回测

```bash
clawquant backtest run ma_crossover --symbol BTC/USDT --interval 1h --days 30
```

这个命令的意思是：
- 用 **ma_crossover**（均线交叉）策略
- 在 **BTC/USDT** 上
- 用 **1小时** K线
- 回测最近 **30天**

## 第三步：看报告

### 生成报告

```bash
clawquant report generate ma_crossover_btc_usdt_20260305_a44a894a
```

（把 run_id 换成你自己的）

### 报告在哪里？

报告会生成在 `runs/<run_id>/` 目录下，包含 5 个文件：

```
runs/ma_crossover_btc_usdt_20260305_a44a894a/
├── report.md        ← 文字版报告（重点看这个！）
├── report.json      ← 结构化数据
├── equity.png       ← 权益曲线图（你的钱是怎么涨跌的）
├── drawdown.png     ← 回撤图（最大亏损是多少）
└── trades.png       ← 交易标记图（在哪里买卖的）
```

### 怎么看报告？

**方法 1: 直接在终端看**

```bash
cat runs/ma_crossover_btc_usdt_20260305_a44a894a/report.md
```

**方法 2: 用 VS Code 或其他编辑器打开**（推荐，Markdown 渲染后更好看）

```bash
code runs/ma_crossover_btc_usdt_20260305_a44a894a/report.md
```

**方法 3: 用 Mac 预览看图表**

```bash
open runs/ma_crossover_btc_usdt_20260305_a44a894a/equity.png
```

### 报告里有什么？

报告是中文的专业分析报告，包含：

1. **概览** — 一眼看出赚了还是亏了，稳定性评分多少
2. **数据可靠性提示** — 如果数据太少会提醒你
3. **绩效指标详解** — 每个指标都有中文解释（Sharpe 是什么、回撤是好是坏）
4. **稳定性评分详解** — 5 个维度的打分和解读
5. **关键发现** — 自动总结策略的优缺点
6. **风险提示与建议** — 告诉你下一步该做什么
7. **交易明细** — 每一笔买卖的时间和盈亏
8. **图表** — 权益曲线、回撤、交易标记

## 进阶：批量对比策略

一次性对比多个策略在多个币种上的表现：

```bash
# 3 个策略 × 2 个币种 = 6 次回测，自动并行执行
clawquant backtest batch dca,ma_crossover,grid \
  --symbols BTC/USDT,ETH/USDT \
  --interval 1h --days 30
```

输出会是一个对比表格，一眼看出哪个策略最好。

## 进阶：参数调优

不知道均线用多少周期最好？让系统自动扫描：

```bash
# 网格搜索：快线 [5,10,20] × 慢线 [20,30,50] = 9 种组合
clawquant backtest sweep ma_crossover \
  --symbol BTC/USDT --days 30 \
  --grid '{"fast_period": [5, 10, 20], "slow_period": [20, 30, 50]}'
```

系统会自动跑完所有参数组合，按收益排名输出结果。

### 随机搜索

参数太多时，网格搜索组合爆炸。可以用随机搜索：

```bash
clawquant backtest sweep ma_crossover \
  --symbol BTC/USDT --days 30 \
  --mode random --n-random 50
```

## 进阶：走前验证

参数调优可能过拟合（在历史数据上表现好，未来不一定好）。走前验证可以检验：

```bash
clawquant backtest walkforward ma_crossover \
  --symbol BTC/USDT --days 90 --splits 3
```

原理：把 90 天的数据分成 3 段，每段先用 70% 训练（找最优参数），再用剩下 30% 测试。如果 3 段测试都盈利，说明策略比较稳健。

## 内置策略介绍

### DCA（定投策略）

**原理**: 每隔固定时间买入固定金额，不管价格高低。适合长期看好某个币种。

```bash
clawquant backtest run dca --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `buy_interval` | 24 | 每隔多少根K线买一次 |
| `buy_amount_usdt` | 100 | 每次买多少 USDT |
| `max_position_usdt` | 50000 | 最大持仓上限 |

### MA Crossover（均线交叉策略）

**原理**: 当短期均线上穿长期均线时买入（金叉），下穿时卖出（死叉）。经典的趋势跟踪策略。

```bash
clawquant backtest run ma_crossover --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `fast_period` | 10 | 快线周期 |
| `slow_period` | 30 | 慢线周期 |
| `ma_type` | SMA | 均线类型（SMA 简单均线 / EMA 指数均线） |
| `position_pct` | 0.1 | 每次用总资金的多少比例交易 |

### Grid（网格交易策略）

**原理**: 在当前价格上下设置多个买卖网格，价格下跌到某个网格就买入，上涨到某个网格就卖出。适合震荡行情。

```bash
clawquant backtest run grid --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `grid_levels` | 5 | 上下各多少个网格 |
| `grid_spacing_pct` | 2.0 | 网格间距（价格百分比） |
| `amount_per_grid` | 200 | 每个网格买卖多少 USDT |

### RSI Reversal（RSI 反转策略）

**原理**: 当 RSI 低于超卖线时买入（认为价格被低估），高于超买线时卖出（认为价格被高估）。经典的均值回归策略。

```bash
clawquant backtest run rsi_reversal --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `rsi_period` | 14 | RSI 计算周期 |
| `oversold` | 30 | 超卖线（低于此值触发买入） |
| `overbought` | 70 | 超买线（高于此值触发卖出） |
| `position_pct` | 0.1 | 每次用总资金的多少比例交易 |

### Bollinger Bands（布林带策略）

**原理**: 价格触及布林带下轨时买入，触及上轨时卖出。布林带由中轨（均线）和上下轨（均线 ± N 倍标准差）组成，利用波动率做均值回归。

```bash
clawquant backtest run bollinger_bands --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `bb_period` | 20 | 中轨均线周期 |
| `bb_std` | 2.0 | 上下轨的标准差倍数 |
| `position_pct` | 0.1 | 每次用总资金的多少比例交易 |

### MACD（指数平滑异同移动平均线策略）

**原理**: 当 MACD 线上穿信号线时买入（动量向上），下穿时卖出（动量向下）。趋势跟踪型策略。

```bash
clawquant backtest run macd --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `fast_period` | 12 | 快线 EMA 周期 |
| `slow_period` | 26 | 慢线 EMA 周期 |
| `signal_period` | 9 | 信号线 EMA 周期 |
| `position_pct` | 0.1 | 每次用总资金的多少比例交易 |

### Breakout（突破策略 / 海龟交易法）

**原理**: 价格突破过去 N 周期最高价时买入，跌破过去 N 周期最低价时卖出。基于唐奇安通道的趋势跟踪策略，源自经典的海龟交易法。

```bash
clawquant backtest run breakout --symbol BTC/USDT --days 90
```

**参数**:
| 参数 | 默认值 | 含义 |
|------|--------|------|
| `lookback` | 20 | 回看周期（取过去多少根K线的高低点） |
| `position_pct` | 0.1 | 每次用总资金的多少比例交易 |

## 自定义策略

想写自己的策略？

### 第一步：生成模板

```bash
clawquant strategy scaffold --name my_awesome_strategy --output ./strategies_user/
```

这会在 `strategies_user/` 目录下生成一个模板文件。

### 第二步：编辑策略

打开生成的文件，填入你的策略逻辑。模板里每个方法都有中英双语注释和示例代码，照着改就行。需要实现 6 个方法：

1. `metadata()` — 策略的名字、版本、参数定义（JSON Schema 格式）
2. `compute_indicators()` — 计算技术指标（如 RSI、MACD），列名必须加策略名前缀
3. `generate_signals()` — 根据指标生成买卖信号，返回 {-1, 0, 1} 的 Series
4. `position_sizing()` — 决定每次买卖多少 USDT，注意不要超过可用现金/持仓
5. `risk_controls()` — 风险控制（如仓位过大时 REDUCE，回撤过大时 FLATTEN）
6. `explain()` — 解释为什么做了上一次操作

**常见错误**:
- `compute_indicators()` 忘记 `df = df.copy()`，导致修改了原始数据
- 指标列名没加策略名前缀（如 `"rsi"` 而非 `"my_strategy_rsi"`），和其他策略冲突
- `generate_signals()` 返回了 float 而不是 int
- 指标 NaN 的行没处理（应该设为 0 = HOLD）
- `position_sizing()` 买入金额超过 `portfolio_state.cash`

### 第三步：验证

```bash
clawquant strategy validate --name my_awesome_strategy
```

### 第四步：回测

```bash
clawquant backtest run my_awesome_strategy --symbol BTC/USDT --days 30
```

## 常见问题

### Q: 拉数据报错 "连接超时"

需要代理。创建 `.env` 文件并填入代理地址：
```
HTTPS_PROXY=http://127.0.0.1:7897
```

### Q: 回测结果显示 Sharpe Ratio 几百甚至几千？

这是因为数据量太少（比如只有 3 天的数据），短期收益被年化后数值爆炸。**不要当真**，报告里会有警告提示。建议用至少 30 天以上的数据。

### Q: 该选哪个K线周期？

- **日线 (`1d`)**: 适合做长期趋势分析，数据量少但稳定
- **4小时 (`4h`)**: 中期分析，平衡了数据量和噪音
- **1小时 (`1h`)**: 最常用，适合大多数策略
- **15分钟 (`15m`)**: 短线策略，需要更多数据
- **5分钟 (`5m`)**: 超短线/高频，手续费影响大

### Q: 该用多长时间的数据？

- **最少**: 30 天（基本可用）
- **建议**: 90 天以上（统计更可靠）
- **理想**: 180-365 天（能覆盖不同市场阶段）

### Q: 回测赚钱 = 实盘也能赚钱？

**不一定。** 回测有几个局限：
1. **过拟合**: 参数可能刚好适合历史数据，未来不一定有效
2. **滑点**: 实盘成交价可能比回测差
3. **流动性**: 大单可能无法按理想价格成交
4. **黑天鹅**: 历史上没出现过的极端行情

**建议**: 先用 `backtest walkforward` 做走前验证，再用 `deploy paper`（模拟交易）观察一段时间，最后再考虑实盘。

### Q: 报告在哪个目录？

`runs/` 目录下，按 run_id 分文件夹：
```bash
ls runs/
# 会看到类似:
# ma_crossover_btc_usdt_20260305_a44a894a/
# dca_btc_usdt_20260305_b55b895b/
```

### Q: 怎么删除旧的回测数据？

```bash
# 删除某次回测
rm -rf runs/ma_crossover_btc_usdt_20260305_a44a894a

# 清空所有回测
rm -rf runs/*

# 清空数据缓存
rm -rf data_cache/*
```

### Q: 支持哪些交易所？

默认使用 Binance。理论上 ccxt 支持的交易所都可以用（100+），但目前只测试了 Binance。

## 名词解释

| 术语 | 含义 |
|------|------|
| **K线/蜡烛图** | 一段时间内的开盘价、最高价、最低价、收盘价和成交量 |
| **回测 (Backtest)** | 用历史数据模拟策略交易，看看过去会赚多少/亏多少 |
| **总收益 (Total Return)** | 回测结束时赚了或亏了多少钱 |
| **最大回撤 (Max Drawdown)** | 从最高点到最低点亏了多少。比如你从 100 涨到 120 再跌到 100，最大回撤就是 16.7% |
| **Sharpe Ratio** | 风险调整后的收益。越高越好。>1 算不错，>2 算优秀。简单理解：每承受 1 单位风险赚了多少 |
| **Sortino Ratio** | 类似 Sharpe，但只考虑下行风险（亏损的波动），更合理 |
| **胜率 (Win Rate)** | 盈利交易占总交易的比例。注意：胜率高不一定赚钱（每笔赚1亏10，胜率90%也亏） |
| **盈亏比 (Profit Factor)** | 总盈利 / 总亏损。>1 说明赚的比亏的多 |
| **年化收益** | 把回测期间的收益换算成一年的收益率。注意短期回测的年化数据可能很夸张 |
| **波动率 (Volatility)** | 收益的波动幅度，越大说明越不稳定 |
| **bps (基点)** | 1 bps = 0.01%。手续费 10 bps 就是 0.1% |
| **金叉/死叉** | 短期均线上穿/下穿长期均线 |
| **DCA (定投)** | Dollar Cost Averaging，定期定额买入 |
| **走前验证 (Walk-forward)** | 把历史数据分成多段，交替训练和测试，检验策略是否真的有效 |
